# 开发计划

## 版本规划总览

### 1.0 MVP 版本目标

实现咖啡点单微信小程序的核心功能，包括用户浏览、点单、购物车、订单提交和历史订单查看。本版本使用 mock 用户数据，暂不实现登录功能，后台管理系统暂不开发，通过数据库初始化脚本提供必要的测试数据。

**核心功能**：
- 首页轮播图展示
- 点单页面（分类和产品展示）
- 购物车功能（前端本地存储）
- 订单确认与提交
- 历史订单查看
- Docker 容器化部署

### 2.0 功能增强版本目标

完善用户认证体系，实现微信授权登录功能，并开发完整的后台管理系统，支持内容管理和订单管理。

**新增功能**：
- 微信小程序用户登录（微信授权 + JWT 认证）
- 后台管理系统（Vue3 + Ant Design Vue）
- 内容管理功能（轮播图、分类、产品的 CRUD）
- 订单管理功能（订单列表查看和状态更新）
- 文件上传与 Nginx 静态资源服务

### 3.0 个性化功能版本目标

在核心功能基础上，增加个性化功能，提升用户粘性和体验，包括积分系统、抽奖系统等创新功能。

**计划功能**：
- 积分系统（用户积分获取、消费、兑换）
- 抽奖系统（积分抽奖、活动抽奖）
- 其他个性化功能（根据业务需求探索）

## 任务依赖关系说明

### 1.0 版本任务顺序

1. **TASK001（项目初始化）** 是基础，所有后续任务都依赖于此
2. **TASK002（首页轮播图）** 和 **TASK003（点单页面）** 可以并行开发，但都依赖 TASK001
3. **TASK004（购物车）** 依赖 TASK003（需要产品数据）
4. **TASK005（订单确认与提交）** 依赖 TASK004（需要购物车数据）
5. **TASK006（历史订单）** 依赖 TASK005（需要订单数据）
6. **TASK007（Docker 部署）** 可以独立进行，但建议在所有功能开发完成后进行

### 2.0 版本任务顺序

1. **TASK008（用户登录）** 是基础，后续任务可能需要用户认证
2. **TASK009（后台管理系统初始化）** 是后台功能的基础
