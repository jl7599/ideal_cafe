# 登录认证与用户体系

## 1. 业务目标

- C端（小程序）：无感登录，减少下单摩擦
- S端/A端（网页版）：安全登录，账号密码认证

## 2. 平台与角色映射

| 平台 | 访问方式 | 角色 | 认证方式 |
|------|---------|------|---------|
| 微信小程序 | 扫码/搜索进入 | 消费者（C端） | 微信授权登录 |
| 网页版 | 浏览器访问管理后台 | 店员（S端）/ 管理员（A端） | 账号密码登录 |

**平台与角色**：小程序仅服务消费者，网页版服务店员和管理员。店员和管理员共用网页版，登录后根据角色跳转对应页面，页面内容根据角色控制可见范围。

## 3. C端 — 微信授权登录（小程序）

### 3.1 登录流程

```
用户打开小程序
    │
    ├─ 已登录且登录状态有效 ──→ 直接进入首页
    │
    └─ 未登录或登录状态过期
         │
         ├─ 静默登录（自动完成，无需用户操作）
         │    │
         │    ├─ 成功 ──→ 进入首页
         │    │
         │    └─ 失败 ──→ 展示登录按钮页面
         │
         └─ 点击登录按钮 ──→ 重新尝试登录 ──→ ...
```

- 首次打开：自动静默登录，无需用户操作
- 获取昵称头像：MVP 不强制获取，用户可在「我的」页面主动设置
- 手机号：MVP 不要求，记为后续迭代

### 3.2 登录状态管理

- 登录成功后，登录状态在本地保存
- 登录状态过期时，自动静默续期
- 续期失败时，重新走登录流程
- 并发操作时，续期请求只发一次，其他操作等待结果

### 3.3 用户信息

首次登录后，用户信息默认值：
- 昵称：默认「理想咖啡」+ 随机4位数字
- 头像：使用默认头像
- 用户可在「我的」页面修改昵称和头像

## 4. S端/A端 — 账号密码登录（网页版）

### 4.1 账号体系

- 店员账号：由管理员在网页版后台创建（手机号 + 密码）
- 管理员账号：初始管理员通过初始化脚本创建，后续管理员可由现有管理员创建

### 4.2 登录流程

```
店员/管理员打开网页版
    │
    ├─ 已登录且登录状态有效 ──→ 直接进入对应首页
    │
    └─ 未登录或登录状态过期
         │
         └─ 展示登录页
              │
              ├─ 输入手机号 + 密码
              ├─ 点击登录
              ├─ 校验通过 ──→ 根据角色跳转对应首页
              └─ 校验不通过 ──→ 行内提示「手机号或密码错误」
```

### 4.3 登录页

- 表单字段：手机号、密码
- 手机号校验：11 位数字，以 1 开头
- 密码校验：6-20 位
- 登录按钮：校验通过后可点击
- 错误提示：行内提示「手机号或密码错误」

### 4.4 登录后跳转

- 店员登录成功 → 跳转订单处理页（S端导航第一个页面）
- 管理员登录成功 → 跳转订单列表页（A端导航第一个页面）

### 4.5 角色与页面可见范围

- 店员：仅可见订单处理页
- 管理员：可见全部页面（订单/商品/门店/店员）
- 后端接口也根据角色校验权限，不依赖前端控制

### 4.6 会话管理

- 登录成功后登录状态保存在浏览器中
- 浏览器关闭后登录状态仍在，下次打开自动进入
- 登录状态过期策略与 C端相同

## 5. 店员账号管理（网页版，A端）

### 5.1 店员列表

- 展示所有店员账号
- 每项展示：手机号、创建时间、状态

### 5.2 新增店员

- 手机号（必填，唯一）
- 密码（必填，6-20 位）

### 5.3 编辑店员

- 可重置密码

### 5.4 删除店员

- 二次确认
- 删除后该店员账号不可登录

## 6. 管理员账号管理

- 管理员可在网页版修改自身密码（需输入旧密码）
- 初始管理员通过初始化脚本创建

## 7. 登出

- C端：在「我的」页面提供「退出登录」入口，清除登录状态
- S端/A端：在网页版右上角头像菜单提供「退出登录」，跳转登录页
- 登出后不会删除用户数据，重新登录可恢复

## 8. C端「我的」页面

- 展示用户昵称和头像
- 提供修改昵称和头像的入口
- 提供退出登录入口
