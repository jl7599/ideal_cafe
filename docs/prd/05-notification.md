# 消息通知

## 1. 业务目标

- 关键节点及时触达用户，减少等待焦虑
- 店员不漏掉新订单

## 2. C端通知场景

| 触发事件 | 通知方式 | 内容摘要 | 优先级 |
|---------|---------|---------|--------|
| 订单被接单 | 页面内提示 | 「您的订单已被接单，预计 XX 分钟后出品」 | P0 |
| 订单制作完成（待取餐） | 页面内提示 | 「您的咖啡已制作完成，请到吧台取餐，取餐码：XXXX」 | P0 |
| 订单被拒绝 | 页面内提示 | 「您的订单已被拒绝，原因：XXX」 | P0 |
| 订单状态变更 | 页面内自动更新 | 进度条节点更新 + 列表状态标签更新 | P0 |

### 2.1 页面内通知实现

- **订单详情页**：自动检测订单状态变更，变更时更新进度条 + 顶部弹出提示条
- **订单列表页**：当前订单状态标签自动更新
- 仅在对应页面活跃时生效，离开页面停止检测

### 2.2 微信订阅消息（后续迭代）

| 通知场景 | 模板名称 | 模板内容 |
|---------|---------|---------|
| 订单被接单 | 接单通知 | {{店名}}：您的订单已接单，预计{{等待时间}}分钟出品 |
| 制作完成 | 取餐通知 | {{店名}}：您的咖啡已制作完成，取餐码{{取餐码}}，请尽快取餐 |
| 订单被拒绝 | 拒绝通知 | {{店名}}：您的订单已被拒绝，原因{{拒绝原因}} |

**订阅消息授权时机**：用户首次提交订单成功后，询问是否接收订单状态通知。

## 3. S端通知场景（网页版）

| 触发事件 | 通知方式 | 内容摘要 | 优先级 |
|---------|---------|---------|--------|
| 新订单到达 | 声音提示 + 页面视觉提示 | 「新订单 #XXXXX」 | P0 |

### 3.1 新订单提示实现

**声音提示**：
- 检测到新订单时播放提示音
- 提示音为短促铃声，播放一次
- 浏览器需用户先与页面交互后声音才生效（页面加载后引导用户点击「开始接单」）
- 可在设置中关闭声音提示

**页面视觉提示**：
- 待处理订单数量角标更新
- 页面顶部弹出提示条「新订单 #XXXXX」，3 秒后自动消失
- 提示条可点击，跳转该订单详情

**新订单检测方式**：
- MVP：定时检测（短间隔），对比待处理订单数量变化
- 后续优化：更实时的推送方案

## 4. 通知规范

### 4.1 提示条样式

- 位置：页面顶部，不遮挡操作区域
- 背景：浅色底 + 左侧图标
- 自动消失：3 秒后自动收起
- 交互：可点击跳转对应详情页

### 4.2 声音提示规范（网页版）

- 时长：1-2 秒
- 音量：跟随系统设置
- 可关闭：网页版设置中提供声音开关
