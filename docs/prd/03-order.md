# 下单与订单全生命周期

## 1. 业务目标

- C端：极简下单流程，自动感知订单进度，历史订单可追溯
- S端：高效处理订单，不漏单不乱序
- A端：可查看全量订单数据

## 2. 订单状态机

### 2.1 状态定义

| 状态 | 名称 | 含义 |
|------|------|------|
| pending | 待处理 | 用户已提交订单，等待店员接单 |
| accepted | 已接单 | 店员确认接单 |
| preparing | 制作中 | 店员开始制作 |
| ready | 待取餐 | 制作完成，等待顾客取餐 |
| completed | 已完成 | 顾客已取餐 |
| cancelled | 已取消 | 用户主动取消 |
| rejected | 已拒绝 | 店员拒绝接单 |

### 2.2 状态流转

```
正向流转：
pending ──→ accepted ──→ preparing ──→ ready ──→ completed

逆向流转：
pending ──→ cancelled（用户取消）
pending ──→ rejected（店员拒绝）
```

流转规则：
- 用户仅在 `pending` 状态下可取消订单
- `accepted` 之后用户不可取消
- 状态只能正向流转（除取消/拒绝外），不可回退，不可跳步
- 店员拒绝订单时，必须填写拒绝原因

### 2.3 各状态下可执行操作

| 状态 | 消费者(C) | 店员(S) | 管理员(A) |
|------|-----------|---------|-----------|
| pending | 取消订单 | 接单、拒绝 | - |
| accepted | - | 开始制作 | - |
| preparing | - | 制作完成 | - |
| ready | - | 确认取餐 | - |
| completed | 再来一单 | - | 查看 |
| cancelled | 再来一单 | - | 查看 |
| rejected | 再来一单 | - | 查看 |

### 2.4 打烊与进行中订单

门店打烊不影响进行中订单（pending/accepted/preparing/ready）的处理和取餐。店员仍可正常操作订单状态流转，用户仍可查看订单状态和取餐码。

## 3. C端 — 下单流程

### 3.1 确认订单页

从购物车点击「去结算」进入。

**页面内容**：
- 订单明细：商品名称、单价、数量、小计
- 整单备注：文本输入框，最多 200 字（选填）
- 金额汇总：
  - 商品合计：¥XX.XX
  - 合计：¥XX.XX（MVP 无额外费用）
- 取餐方式：自取（默认且唯一选项，灰色展示）
- 「提交订单」按钮（显示合计金额）

### 3.2 提交订单

1. 点击「提交订单」
2. 弹出确认弹窗：「确认提交订单 ¥XX.XX？」，含「取消」和「确认提交」两个按钮
3. 用户点击「确认提交」→ 订单创建成功，状态为 `pending`
4. 跳转订单详情页

**防重复提交**：短时间内重复点击只会创建一个订单。

**金额校验**：订单金额以后台商品价格为准，不依赖前端传递的金额。

### 3.3 订单状态追踪

**订单详情页**展示订单状态进度条：

```
待处理 ── 已接单 ── 制作中 ── 待取餐 ── 已完成
  ●        ○        ○        ○        ○
```

- 当前状态及之前节点高亮实心
- 当前状态之后节点灰色空心
- 状态变更时进度条自动更新

**取餐码**：
- 订单创建时即生成 4 位数字取餐码
- 订单进入 `ready` 状态时，订单详情页大字展示取餐码，便于识别

**状态自动更新**：
- 订单详情页自动检测状态变更（短时间内生效）
- 订单列表页的当前订单状态标签也自动更新
- 离开页面后停止检测

### 3.4 订单列表页（「订单」Tab）

**两个区域**：

1. **当前订单**：状态为 pending/accepted/preparing/ready 的订单，按时间倒序
2. **历史订单**：状态为 completed/cancelled/rejected 的订单，按时间倒序，分页加载（每页 20 条）

**列表项展示**：
- 订单号（后 6 位）
- 下单时间
- 商品摘要（如「燕麦拿铁 x1, 美式 x2」）
- 合计金额
- 状态标签（不同状态不同颜色）

**点击列表项** → 进入订单详情页

### 3.5 再来一单

在已完成/已取消/已拒绝的订单详情页提供「再来一单」按钮：
- 将该订单的所有商品加入购物车
- 如果某商品已下架，跳过该商品并 Toast 提示「部分商品已下架」
- 如果门店已打烊，加入购物车后 Toast 提示「门店已打烊，暂不可下单」
- 跳转购物车页面

### 3.6 用户取消订单

- 仅 `pending` 状态的订单可取消
- 点击「取消订单」→ 弹出取消原因输入框（选填）→ 确认取消
- 订单状态变为 `cancelled`
- 取消后订单详情展示取消原因

### 3.7 商品快照

订单创建时保存商品快照（商品名称、价格），商品后续被删除或改价不影响已有订单的展示。

## 4. S端 — 订单处理（网页版）

### 4.1 订单列表

**三个 Tab**：

| Tab | 包含状态 | 说明 |
|-----|---------|------|
| 待处理 | pending | 新订单，需要尽快处理 |
| 进行中 | accepted, preparing, ready | 正在处理的订单 |
| 已完成 | completed, cancelled, rejected | 已结束的订单 |

**列表项展示**：
- 订单号（后 6 位）
- 下单时间
- 商品摘要
- 合计金额
- 状态标签
- 操作按钮（根据状态动态显示）

**新订单提示**：
- 「待处理」Tab 角标显示待处理订单数量
- 新订单到达时：页面顶部弹出提示条 + 声音提醒
- 网页版需用户先点击页面（如「开始接单」按钮）后，声音提示才生效
- 声音提醒可在设置中关闭

### 4.2 订单详情与操作

**操作按钮根据状态动态显示**：

| 当前状态 | 可执行操作 | 按钮文案 |
|---------|-----------|---------|
| pending | 接单 / 拒绝 | 「接单」「拒绝」 |
| accepted | 开始制作 | 「开始制作」 |
| preparing | 制作完成 | 「制作完成」 |
| ready | 确认取餐 | 「确认取餐」 |
| completed | - | 无操作按钮 |
| cancelled | - | 无操作按钮，展示取消原因 |
| rejected | - | 无操作按钮，展示拒绝原因 |

**拒绝订单**：
- 点击「拒绝」后弹出输入框，填写拒绝原因（必填）
- 确认后订单状态变为 `rejected`
- 拒绝原因展示在订单详情中

**接单时预估等待时间**（选填）：
- 接单弹窗中可选填预估等待时间（分钟）
- 填写后展示在 C端订单详情中

### 4.3 订单详情页

- 完整订单信息：订单号、下单时间、商品明细（名称/单价/数量/小计）、整单备注、合计金额
- 当前状态
- 操作按钮区
- 若已取消，展示取消原因
- 若已拒绝，展示拒绝原因

## 5. A端 — 订单查看（网页版）

### 5.1 订单列表

- 展示全量订单
- 筛选条件：日期范围（开始日期-结束日期）、订单状态（下拉多选）
- 列表项展示：订单号、下单时间、用户昵称、商品摘要、金额、状态
- 按时间倒序排列
- 分页加载

### 5.2 订单详情

- 查看订单完整信息（与 S端相同）
- 无操作按钮（A端仅查看，不操作）

## 6. 订单号生成规则

- 格式：`yyyyMMddHHmmss` + 4 位随机数字
- 示例：`202604091430258827`
- 全局唯一
- 列表展示使用订单号后 6 位
