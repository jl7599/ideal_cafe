# 门店信息与配置

## 1. 业务目标

- C端：顾客了解门店基本信息和营业状态，打烊时无法下单
- A端：管理员可维护门店信息，灵活控制营业状态

## 2. 门店信息项

| 属性 | 说明 | 示例 |
|------|------|------|
| 店名 | 门店名称 | Ideal Cafe |
| 地址 | 门店地址 | XX 市 XX 区 XX 路 XX 号 |
| 联系电话 | 门店电话 | 138-XXXX-XXXX |
| 营业时间 | 按星期设置每天的开闭店时间 | 周一至周五 08:00-20:00，周六日 09:00-21:00 |
| 公告 | 门店公告内容 | 「今日新品：埃塞俄比亚水洗耶加雪菲」 |
| 营业状态 | 当前是否营业 | 营业中 / 已打烊 |
| 门店 Logo | 门店 Logo 图片 | URL |

**单店模型**：全局仅一条门店记录，所有配置针对这一家店。

## 3. C端 — 门店信息展示

### 3.1 门店信息展示位置

- 「我的」页面顶部展示门店基本信息卡片（店名、地址、营业时间、电话）
- 菜单页顶部显示营业状态标识

### 3.2 营业状态

**状态判定**：
- 营业中：管理员手动设置为营业 + 当前时间在营业时间范围内
- 已打烊：管理员手动设置为打烊 或 当前时间不在营业时间范围内

**营业中的展示**：
- 菜单页顶部绿色标识「营业中」
- 下单流程正常可用

**已打烊的展示**：
- 菜单页顶部灰色标识「已打烊」
- 菜单页仍可浏览，但商品卡片不展示「加入购物车」入口
- 购物车页「去结算」按钮置灰，提示「门店已打烊，暂不可下单」

### 3.3 门店公告

- 公告内容在菜单页顶部滚动展示（跑马灯效果）
- 无公告时菜单页不展示公告栏
- 公告栏位于营业状态标识下方

### 3.4 营业状态变化感知

- C端菜单页和购物车页需感知营业状态变化
- 管理员切换营业状态后，C端在短时间内生效
- 打烊时不影响进行中订单的处理和取餐

## 4. A端 — 门店配置（网页版）

### 4.1 基本信息

- 编辑店名（必填）
- 编辑地址（必填）
- 编辑联系电话（选填）
- 上传门店 Logo（选填）

### 4.2 营业时间

按星期设置，每天的配置项：

| 配置项 | 说明 |
|--------|------|
| 是否营业 | 当天是否营业（如周一休息） |
| 开店时间 | 格式 HH:mm，如 08:00 |
| 闭店时间 | 格式 HH:mm，如 20:00 |

- 默认值：周一至周日全部 08:00-20:00
- 支持一键设置「全部相同」：将某一天的配置应用到所有天

### 4.3 营业状态切换

- 开关控件：营业中 ↔ 已打烊
- 手动打烊可覆盖营业时间判定（如临时休息）
- 手动打烊时提示「打烊后顾客将无法下单，确认打烊？」
- 手动恢复营业后，营业状态重新按营业时间自动判定

### 4.4 公告管理

- 公告内容：文本输入，最多 500 字
- 启用/禁用开关
- 禁用后 C端不展示公告栏
