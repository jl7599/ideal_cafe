# 工作计划设计

## 目标

基于 `docs/design/` 的详细设计文档，制定最小粒度、可快速验证的工作计划，写入 `docs/plan/` 目录，供前端/后端/测试/部署团队领取任务。

## 计划维度

**按里程碑+模块**：7 个里程碑，每个里程碑内按角色拆任务（后端/C端/S-A端/测试/部署）。

## 里程碑划分

```
M1(脚手架) → M2(认证) → M3(菜单) ──→ M5(订单) → M6(通知) → M7(部署)
                    └→ M4(门店) ──┘
```

M3 和 M4 可并行。

| 里程碑 | 名称 | 后端 | C端 | S/A端 | 测试 | 部署 | 合计 |
|--------|------|------|-----|-------|------|------|------|
| M1 | 项目脚手架 & 数据库 | 8 | 4 | 4 | 1 | 3 | 20 |
| M2 | 认证模块 | 11 | 4 | 3 | 1 | - | 19 |
| M3 | 菜单模块 | 7 | 3 | 3 | 3 | - | 16 |
| M4 | 门店模块 | 7 | 2 | 3 | 1 | - | 13 |
| M5 | 订单模块 | 15 | 5 | 3 | 3 | - | 26 |
| M6 | 通知/轮询 | - | 4 | 3 | 1 | - | 8 |
| M7 | 部署上线 | - | - | - | 1 | 5 | 6 |
| **合计** | | **48** | **22** | **19** | **11** | **8** | **108** |

```
├── 03-shop.md             # M4
├── 04-order.md            # M5
├── 05-notification.md     # M6

```

## 每个计划文件格式

```markdown

## 目标
一段话

## 前置依赖

## 任务清单

### 后端任务
| 任务ID | 描述 | 验收标准 | 复杂度 |

### C端前端任务
...

### S/A端前端任务
...

### 测试任务
...

### 部署任务
...

## 里程碑验收条件
- [ ] ...
```

## 复杂度定义

- **S** (Small): 0.5-1 天，逻辑简单
- **M** (Medium): 1-2 天，有业务逻辑或交互
- **L** (Large): 2-3 天，复杂业务逻辑或多组件联动
